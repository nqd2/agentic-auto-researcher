Dựa vào file `payload.json` bạn vừa cung cấp, đây là một request từ hệ thống **Cursor IDE** gửi đến mô hình LLM (cụ thể là `gpt-4o`). Payload này chứa toàn bộ system prompt, context và bộ tools đồ sộ mà Cursor trang bị cho AI của họ.

Dưới đây là các đặc điểm nổi bật của payload này và bảng so sánh chi tiết với **Claude Code**. (Và tất nhiên, tuân thủ yêu cầu không mở canvas nhé!).

### 1. Đặc điểm cốt lõi của Cursor Payload

* **Bơm Context (Ngữ cảnh) cực kỳ dày đặc:**
    * Cursor tự động đính kèm thông tin về trạng thái IDE của người dùng: hệ điều hành, shell, đường dẫn workspace, trạng thái Git hiện tại (các file đang bị sửa đổi, chưa track), thư mục terminal chứa output của các lệnh.
    * Nó còn đính kèm danh sách "Các file đang mở và xem gần đây" (`<open_and_recently_viewed_files>`) để AI biết người dùng đang quan tâm đến phần nào của dự án.
* **Hệ thống Trích dẫn Code (Code Citation) nghiêm ngặt để render UI:**
    * Cursor ép AI phải dùng một cú pháp đặc biệt để trích dẫn code có sẵn: 
* **Hệ thống Skills và Rules động:**
    * Payload chèn vào một danh sách dài các `<available_skills>` (như các file Markdown hướng dẫn dùng GitNexus, API design, Node.js best practices, v.v.). AI được yêu cầu dùng công cụ `Read` để đọc các file này khi cần thiết.
    * Nó cũng đọc các rules người dùng tự định nghĩa (ví dụ: "Always answer in Vietnamese").
* **Hệ thống Tools khổng lồ và tích hợp MCP/Extension:**
    * **Core Tools:** `Shell`, `Grep`, `Glob`, `Read`, `StrReplace`, `EditNotebook`, `ReadLints`.
    * **Task & Sub-agents:** Công cụ `Task` cho phép đẻ ra các agent con với các role chuyên biệt (như `explore`, `shell`, `browser-use`, `best-of-n-runner`).
    * **UI Tracking:** Công cụ `TodoWrite` để AI tự tạo checklist hiển thị lên giao diện cho người dùng theo dõi tiến độ.
    * **Third-party/MCP Tools:** Tích hợp sâu các tool từ GitLens (quản lý PR, commit), GitNexus (truy vấn đồ thị mã nguồn, impact analysis), và Stitch (hệ thống sinh UI/Design System).
    * **Browser:** Có nguyên một bộ tool `cursor-ide-browser-*` để tự động hóa duyệt web, click, fill form, chụp ảnh màn hình (phục vụ test UI).

### 2. So sánh Cursor (dựa trên payload) và Claude Code

Mặc dù cả hai đều là AI Coding Assistant, triết lý thiết kế của chúng hoàn toàn khác nhau. Dưới đây là bảng so sánh:

| Tiêu chí | Cursor (Dựa trên Payload) | Claude Code (CLI) |
| :--- | :--- | :--- |
| **Môi trường hoạt động** | Tích hợp sâu vào IDE (giao diện đồ họa, panel). | Hoạt động chủ yếu qua Terminal/CLI. |
| **Nhận thức Ngữ cảnh (Context)** | Biết chính xác file nào đang mở, con trỏ chuột ở đâu, trạng thái git, terminal logs. | Dựa vào thư mục hiện tại (`cwd`) và các lệnh user chủ động cung cấp. |
| **Cách AI chỉnh sửa Code** | Dùng `StrReplace` (thay thế chuỗi chính xác) và cú pháp `startLine:endLine` để IDE render UI apply code. | Dùng các tool như `grep`, `bash`, `read_file`, `write_file` hoặc in ra file diff. |
| **Quản lý Task/Tiến độ** | Dùng công cụ `TodoWrite` để tạo checklist trạng thái (pending, in_progress, completed) hiển thị lên UI. | Ghi markdown ra file kế hoạch (Ví dụ: `.aar/plans` hoặc dùng text thuần túy). |
| **Quản lý Sub-agent** | Gọi công cụ `Task` để ủy quyền cho các agent chuyên biệt (`browser-use`, `explore`, v.v.) chạy ngầm. | Cũng có Sub-agent/Task nhưng thường phục vụ chạy script hoặc lệnh bash tốn thời gian. |
| **Hệ sinh thái Công cụ (Tools)** | Rất đồ sộ. Bao gồm tools của IDE, tích hợp sẵn MCP (Browser) và các công cụ từ Extension (GitLens, Stitch, GitNexus). | Tinh gọn hơn. Tập trung vào thao tác file, bash, và kết nối MCP Server do người dùng tự config. |
| **Tương tác Đa phương thức** | Có công cụ `GenerateImage` để sinh ảnh, `EditNotebook` chuyên sửa file `.ipynb`, và đọc giao diện web qua ảnh chụp. | Chủ yếu là text in/text out, đọc/ghi file mã nguồn. |
| **Điều hướng Luồng công việc** | AI có thể tự động chuyển chế độ (Plan, Ask, Agent) qua công cụ `SwitchMode` tùy theo độ phức tạp của câu hỏi. | Người dùng dùng lệnh slash (`/plan`, `/bug`) để điều hướng AI. |

**Tóm lại:**
Payload này cho thấy **Cursor** hành xử như một "Bậc thầy thao túng IDE". Nó không chỉ sinh code mà còn giao tiếp liên tục với giao diện của IDE (tạo checklist, đổi chế độ, render block code theo dòng). Trong khi đó, **Claude Code** (hay như dự án Agentic Auto Researcher bạn đang xem xét) có xu hướng giống một "kỹ sư dòng lệnh" độc lập, tinh gọn, ít phụ thuộc vào UI đồ họa và tập trung vào thao tác file/shell truyền thống.
